PhysAgent combines a TCAD physics ontology and literature-driven domain knowledge to guide LLM-based SDevice file generation. The constructed ontology is publicly available as an interactive web platform~\cite{ontology_tcad}.

\subsection{TCAD Physics Ontology}
We construct a directed graph from the Sentaurus TCAD guide with three node types (\textit{equation}, \textit{parameter}, \textit{constant}) and three edge types (\textit{input}, \textit{specialCase}, \textit{pairwise}), capturing inter-model dependencies such as bandgap narrowing affecting intrinsic carrier density and recombination rates ({\color{red}Table~\ref{tab:ontology_mapping}}).

\subsection{Ontology-Guided Generation}
Given a target device and material, PhysAgent accesses the ontology through an MCP (Model Context Protocol) interface that exposes three classes of tools: one returning a node/edge map of the graph, one returning SDevice keywords and syntax for a selected node, and one returning material-specific parameter values together with their source citations drawn from a curated set of TCAD publications (e.g., Masetti, Klaassen, Roschke, Pernot). Since the full ontology spans hundreds of thousands of tokens, direct prompt injection is impractical; the LLM instead follows a selective retrieval loop---reading the overview, fanning out parallel syntax queries for the relevant models, pulling references for parameter values, and composing the Physics block and \texttt{.par} file. This shrinks per-query context by roughly {\color{red}6$\times$} relative to a full-ontology dump while preserving model coverage, and each parameter in the generated \texttt{.par} inherits the citation returned by the reference tool, enabling parameter-level provenance ({\color{red}Fig.~\ref{fig:framework}}).

\subsection{Ontology Expansion and Validation}
Unlike SFT-based approaches requiring retraining for new models or materials, PhysAgent uses a directional synchronization workflow with three artifact layers---Spec, Plan, and Impl---where changes propagate automatically from the source of truth while reverse modifications require explicit approval, ensuring incremental expansion without retraining.
