\subsection{TCAD Physics Ontology}
We construct a physics ontology from the Sentaurus TCAD guide as a directed graph with three node types (\textit{equation}, \textit{parameter}, \textit{constant}) and three edge types (\textit{input}, \textit{specialCase}, \textit{pairwise}). This ontology captures inter-model dependencies---for example, how bandgap narrowing models affect intrinsic carrier density, which in turn influences recombination rates---providing a structured map that would otherwise remain implicit in simulator manuals.

\subsection{Literature-Driven Domain Knowledge}
The ontology captures model structure but not material-specific expertise. PhysAgent supplements it with literature-driven knowledge---validated model combinations and calibrated parameters for specific materials (e.g., SiC, GaN)---bridging the gap between general simulator documentation and practical modeling practice.

\subsection{Ontology-Guided Generation}
Given a target device and material, PhysAgent traverses the ontology to extract a relevant subgraph of models and parameters, combines it with literature knowledge, and provides this as structured context to the LLM. This explicit grounding reduces hallucination and produces physically justified SDevice configurations.
